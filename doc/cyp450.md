# CYP450 RVS 修改策略 Roadmap

# Phase 1 — 停止錯誤 weighting

## Step 1. 移除 global halogen boost

刪除：

```python
score += halogen_count * mw
```

原因：

* Halogen 不具有 target specificity
* 容易污染 kinase / NR / HDAC
* 屬於 promiscuity feature

---

# Phase 2 — 將 CYP450 改成 conditional scoring

## Step 2. 建立 CYP450 conditional motif

不要：

```text
Halogen
Imidazole
Phenyl
```

單獨加分。

改成：

```text
(Imidazole + hydrophobic tail)
(Halogenated phenyl + oxidation site)
(Heteroaromatic N + lipophilic cluster)
```

---

## Step 3. 加入 topology requirement

新增：

* aromatic burial
* ligand elongation
* accessible heteroatom
* planar insertion geometry

避免只有 FG overlap。

---

# Phase 3 — 加入 physicochemical layer

## Step 4. 加入 shape descriptors

新增：

* PMI
* radius of gyration
* asphericity
* graph diameter

目的：

```text
CYP450 → elongated hydrophobic ligand
GPCR → compact amine ligand
```

---

## Step 5. 加入 flexibility descriptors

新增：

* rotatable bond count
* torsion entropy

目的：

區分：

```text
Rigid kinase scaffold
vs
Flexible CYP450-compatible ligand
```

---

# Phase 4 — 加入 metabolism logic

## Step 6. 建立 oxidation soft spot detector

偵測：

* benzylic carbon
* allylic position
* terminal methyl
* exposed alkyl chain

這些比 halogen 更重要。

---

## Step 7. 計算 oxidation accessibility

加入：

* SASA exposure
* steric accessibility
* neighboring bulky groups

避免 inaccessible oxidation site 被誤判。

---

# Phase 5 — 加入 negative constraints

## Step 8. kinase suppression rule

若出現：

```text
flat hinge geometry
+
dual HBA/HBD pair
+
rigid aromatic scaffold
```

則降低 CYP450 score。

---

## Step 9. HDAC suppression rule

若出現：

```text
Zn chelation geometry
```

則 suppress CYP450。

---

# Phase 6 — 修改 scoring architecture

## Step 10. 分離 rarity 與 importance

不要：

```python
score = IDF
```

改成：

```python
score =
importance_weight
*
rarity_weight
```

---

## Step 11. GPCR motif saturation

避免：

```text
amine + aromatic
```

無限累加。

改成 diminishing return：

```python
first_match = 1.0
second_match = 0.3
third_match = 0.1
```

---

# Phase 7 — 長期架構升級

## Step 12. 從 FG matcher 升級為 interaction engine

最終 architecture：

```text
FG
+
geometry
+
topology
+
shape
+
physicochemical compatibility
+
negative constraints
+
target hierarchy
```

---

## Highest ROI

1. 移除 halogen global boost
2. conditional motif
3. negative constraints
4. oxidation soft spot
5. shape descriptors

---

## Medium ROI

6. flexibility descriptors
7. topology scoring

## Low ROI
